# Power-Workflow-Next 产品规格说明书

## 一、概述

### 1.1 项目背景

`power-workflow-next` 是 PowerJob 工作流可视化组件的下一代版本，用于替代基于 @antv/g6 的旧版 `power-workflow` 组件。新版本采用 React Flow 作为基础框架，提供更现代化的开发体验和更美观的视觉效果。

### 1.2 项目目标

| 目标 | 描述 |
|------|------|
| 视觉升级 | 采用扁平简约设计风格，提升用户体验 |
| 技术现代化 | 使用 React 18 + TypeScript + React Flow 12 重构 |
| 降低维护成本 | 利用 React Flow 生态，减少自研代码量 |
| 独立发布 | 作为独立 npm 包，支持多项目复用 |

### 1.3 非目标

- ❌ 不兼容旧版 API（全新设计）
- ❌ 不支持大规模节点场景（50+ 节点）
- ❌ 不支持暗色主题
- ❌ 不新增节点类型

## 三、功能规格

### 3.1 功能优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 节点拖拽与连线 | 核心编辑能力 |
| P0 | 视图模式状态展示 | 运行状态可视化 |
| P1 | 自动布局 | 一键整理节点位置 |
| P1 | 撤销/重做 | 操作回退能力 |
| P2 | 小地图/缩略图 | 大画布导航辅助 |
| P3 | 节点搜索/筛选 | 快速定位节点 |

### 3.2 运行模式

组件支持两种运行模式：

| 模式 | 说明 | 典型场景 |
|------|------|----------|
| `edit` | 编辑模式 | 工作流编辑器 |
| `view` | 查看模式 | 实例详情页、历史记录查看 |

### 3.3 节点类型

保持三种节点类型不变：

| 类型 | Code | 形状 | 用途 |
|------|------|------|------|
| JOB | 1 | 矩形 | 执行具体任务 |
| DECISION | 2 | 菱形 | 条件判断，控制流程分支 |
| NESTED_WORKFLOW | 3 | 矩形（带图标） | 嵌套引用其他工作流 |

### 3.4 视图模式能力

在 `view` 模式下必须支持：

- [x] 点击节点触发详情回调
- [x] 实时状态轮询/刷新（外部控制，组件被动响应）
- [x] 运行动画（边框呼吸效果）
- [x] 缩放/平移浏览

## 九、交互规格

### 8.1 连线规则

| 规则 | 行为 |
|------|------|
| 禁止自连接 | 节点不能连接到自身 |
| 禁止重复连线 | A→B 之间只能有一条边 |
| 任意端口连接 | 不限制输入/输出端口方向 |

**连线创建后：**
- 默认不设置 `property` 属性（即普通连线，灰色无标签，值为 `''`）
- 支持后续点击编辑（设置 `'true'`/`'false'`）

### 8.1.1 连线 property 编辑（Inline 切换）

编辑模式下，点击连线后可通过线上标签 Inline 切换 `property` 值：

| 当前状态 | 点击后状态 | 视觉变化 |
|----------|------------|----------|
| 无标签（property = `''`） | Y（property = `'true'`） | 线变绿，出现 "Y" 标签 |
| Y（property = `'true'`） | N（property = `'false'`） | 线变红，标签改为 "N" |
| N（property = `'false'`） | 无标签（property = `''`） | 线恢复灰色，标签消失 |

**交互细节：**
- 点击连线本身（非标签区域）：选中连线，不触发切换
- 点击连线上的标签（"Y"/"N" 区域）：触发 property 循环切换
- 无标签时：点击连线后，在线的中点出现一个可点击的 `+` 或 `○` 交互区域，点击触发切换
- 切换操作支持撤销/重做

**连线的 `enable` 字段：** 暂不在 UI 中暴露编辑入口，仅作为数据字段保留。

### 8.2 撤销重做范围

| 操作 | 支持撤销 |
|------|----------|
| 节点位置移动 | ✅ |
| 添加/删除节点 | ✅ |
| 添加/删除连线 | ✅ |
| 节点参数修改 | ✅ |
| 连线属性修改 | ✅ |

**配置：** 撤销步数上限可通过 `undoableActions` 配置，默认 50。

**多选时编辑面板：** 关闭编辑面板（多选状态无法编辑单节点参数）。

## 十、数据校验

### 9.1 校验时机

采用双重校验机制：

| 校验类型 | 触发时机 | 校验范围 | 阻断行为 |
|----------|----------|----------|----------|
| **字段级校验** | 离开输入框时 | 当前字段 | 仅提示，不阻止保存 |
| **完整校验** | 点击保存时 | 所有必填和格式 | 有错误时禁用保存按钮 |

工作流级校验（获取工作流数据时触发）：
- 循环依赖检测
- 判断节点无出口检测

### 9.2 节点编辑校验规则

#### 9.2.1 必填字段校验

| 字段 | 适用节点类型 | 校验规则 |
|------|--------------|----------|
| 节点名称 | 所有节点 | 必填，长度 1-50 字符 |
| 判断条件 | 判断节点 | 必填，符合表达式格式 |
| 目标工作流 | 嵌套工作流节点 | 必填，工作流 ID 必须存在 |

#### 9.2.2 格式校验

| 字段 | 校验要求 |
|------|----------|
| 节点名称 | 支持中文、英文、数字、下划线、中划线 |
| 判断条件 | 必须包含比较操作符（等于/不等于/大于/小于/包含/开头/结尾） |
| 节点参数 | 必须是有效的 JSON 格式 |

#### 9.2.3 范围校验（警告）

| 字段 | 校验范围 | 警告条件 |
|------|----------|----------|
| 超时时间 | 1-3600 秒 | 超过 300 秒时提示性能问题 |

### 9.3 工作流级校验规则

| 规则 | 检测内容 | 提示方式 | 提示文案 |
|------|----------|----------|----------|
| 循环依赖 | 检测是否存在闭环 | Toast | 工作流中存在循环依赖，请检查连线 |
| 判断节点无出口 | 检测是否缺少分支连线 | Toast | 判断节点「xxx」缺少出口连线 |
| 孤立节点 | 不校验，允许存在 | - | - |

### 9.4 校验状态提示

#### 9.4.1 字段级提示

| 状态 | 视觉表现 | 提示位置 |
|------|----------|----------|
| 有效 | 默认边框样式 | 无提示 |
| 错误 | 红色边框，下方红色错误文字 | 输入框正下方 |
| 警告 | 橙色边框，下方橙色警告文字 | 输入框正下方 |
| 校验中 | 蓝色边框 | 无提示 |

#### 9.4.2 保存时错误汇总

当点击保存按钮且存在错误时，面板底部显示错误汇总：

```
┌────────────────────────────────┐
│ 💥 校验失败                      │
│ 发现 N 个错误：                   │
│  • 节点名称不能为空               │
│  • 条件表达式格式错误             │
├────────────────────────────────┤
│         [取消]  [保存(禁用)]    │
└────────────────────────────────┘
```

#### 9.4.3 保存时警告确认

当只有警告没有错误时，显示确认弹窗：

```
┌────────────────────────────────┐
│ ⚠️ 保存确认                      │
│ 发现 N 个警告，是否继续保存？     │
│  • 超时时间较大，可能影响性能      │
│                                  │
│   [取消保存]    [继续保存]      │
└────────────────────────────────┘
```

### 9.5 校验交互流程

1. **聚焦输入框**：不显示校验状态，避免干扰输入
2. **输入中**：显示蓝色边框表示正在编辑
3. **离开输入框**：触发字段级校验，即时显示错误或警告
4. **点击保存**：
   - 有错误：自动滚动到第一个错误字段并聚焦，显示错误汇总
   - 有警告：弹出确认对话框
   - 无错误无警告：保存成功，关闭编辑面板

#### 11.1.5 视图模式刷新机制

组件**不内置轮询**，状态刷新完全由外部控制。使用方通过定期更新 `nodes` prop 来驱动组件重渲染：

```tsx
// 推荐用法：外部定时拉取最新状态，更新 nodes 触发组件刷新
const [nodes, setNodes] = useState(initialNodes);

useEffect(() => {
  if (mode !== 'view') return;
  const timer = setInterval(async () => {
    const latestNodes = await fetchNodeStatuses(instanceId);
    setNodes(latestNodes);
  }, 3000); // 外部控制轮询间隔
  return () => clearInterval(timer);
}, [mode, instanceId]);

<WorkflowNext mode="view" nodes={nodes} edges={edges} />
```

**组件行为：**
- `nodes` prop 更新时，组件自动 diff 并更新各节点状态（状态色、呼吸动画等）
- 不需要调用任何 Ref 方法，响应式更新
- 查看模式工具栏**不提供刷新按钮**，手动刷新由外部调用方自行实现

### B. 状态码对照表

> **注意：** 实例状态码与节点状态码是两套独立体系，数值含义不同，勿混淆。

```typescript
// 工作流实例整体状态
enum WorkflowInstanceStatus {
  WAITING  = 1,   // 等待调度
  RUNNING  = 2,   // 运行中
  FAILED   = 3,   // 失败
  SUCCEED  = 4,   // 成功
  STOPPED  = 10,  // 手动停止
}

// 工作流内单个节点执行状态（与实例状态码不同）
enum NodeStatus {
  WAITING  = 1,   // 等待中（尚未执行）
  RUNNING  = 3,   // 运行中（注意：值为 3，与实例的 FAILED=3 不同）
  FAILED   = 4,   // 失败
  SUCCEED  = 5,   // 成功
  STOPPED  = 10,  // 已停止
}
```

### C. 设计决策记录

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 节点尺寸 | 固定尺寸 | 整齐划一，布局可预测 |
| 字体大小 | 14px/12px | 黄金字号，清晰可读 |
| 锚点显示 | 始终显示 | 直观，无需 hover 发现 |
| 连接线型 | 贝塞尔曲线 | 视觉柔和，现代 UI 风格 |
| 分支颜色 | 红绿区分 | 最直观的是/否视觉语言 |
| 国际化 | 内置中英文 | PowerJob 主要用户在国内 |
